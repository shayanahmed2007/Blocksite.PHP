What is a DOM?

The DOM is a Web API that allows developers to use programming logic to make changes to their HTML code. It's a reliable way to make changes that turn static websites into dynamic ones. It's an important topic in web development because the DOM serves as the initial use of PHP in the server. HTML code isn't considered part of the DOM until its parsed by the server. To see what happens to your HTML code when this parsing happens, copy your code from the body tag and paste it here (within the box with the title 'Markup to test' after the three dots).

Core Concepts

In order to effectively understand the DOM and how it relates to working with the web, it is necessary to have an existing knowledge of HTML and CSS. It is also beneficial to have familiarity with fundamental PHP syntax and code structure.

Skill Quest

At the most basic level, a website consists of an HTML document. The browser that you use to view the website is a program that interprets HTML and CSS and renders the style, content, and structure into the page that you see. In addition to parsing the style and structure of the HTML and CSS, the browser creates a representation of the document known as the Document Object Model. This model allows PHP to access the text content and elements of the website document as objects. PHP is an interactive language, and it is easier to understand new concepts by doing. Let's create a very basic website. Create an index.html file and save it in a new project directory.

        
            !DOCTYPE html
 html lang="en"
    head
        titleLearning the DOM/title
    /head
    body
        ?php
            // PHP code to dynamically set the content
            $heading = 'Updated Document Object Model';
        ?>
        h1?php echo $heading; ?>/h1?php>
    /body
  html

This code is the familiar HTML source of a new website skeleton. It contains the absolute most essential aspects of a website document — a doctype, and an html tag with the head and body nested inside.

For our purposes, we'll be using the Chrome browser, but you can receive similar output from other modern browser. Within Chrome, open up index.html. You'll see a plain website with our heading saying “Document Object Model”. Right click anywhere on the page and select “Inspect”. This will open up Developer Tools.

Under the Elements tab, you'll see the DOM.

What is the Difference Between the DOM and HTML Source Code?

Currently, with this example, it seems that HTML source code and the DOM are the exact same thing. There are two instances in which the browser-generated DOM will be different than HTML source code:

  • PHP generates HTML on the server, which is then sent to the client. Any modifications to the DOM must be done using client-side JavaScript after the HTML is rendered in the browser.

  • If there are errors in the HTML generated by PHP, the browser attempts to handle and render the page. PHP itself does not correct HTML errors; it outputs the HTML code as written.

Let's demonstrate how HTML can be dynamically generated by server-side PHP. Add the following code to your PHP file:

document.body;

The echo will respond with this output:

Output

            
                body>
                    h1>Document Object Model/h1>
                  body>

The $_SERVER is a superglobal array, and $_SERVER['DOCUMENT_ROOT'] is an element of that array that we access using array notation. Using $_SERVER['DOCUMENT_ROOT'] in a script will output the root directory of the document. In PHP, we can generate or modify the output content dynamically. For instance, we can generate HTML content that includes specific styles. Here's how you can use PHP to modify content dynamically. For example, to simulate changing the background color dynamically, you can use PHP to include a style directly in the generated HTML.

                    
                        $backgroundColor = 'fuchsia'

After typing and submitting the above code, you'll see the live update to the site, as the background color changes.

Switching to the Elements tab, or typing document.body into the echo again, you will see that the DOM has changed.


        
            output
            body style="background-color: fuchsia;">
                h1Document Object Model/h1>
            /body>

The PHP code we typed, assigning fuchsia to the background color of the body, is now a part of the DOM.
