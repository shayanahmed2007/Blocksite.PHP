What is Sychronous Code?


When we write a program in PHP, it executes line by line. When a line is completely executed,then and then only does the code move forward to execute the next line.


Let's look an example of this:

    
        ?php
$greet_one = "Hello";
$greet_two = "world!!!";
echo $greet_one . "\n"; // Output: Hello
for ($i = 0; $i 1000000000; $i++) {
    // Loop with no operation, similar to the PHP code
}
echo $greet_two . "\n"; // Output: world!!!
?>

Now if you run the above example on your machine you will notice that greet_one logs first.Then the program waits for a couple of seconds and then logs greet_two.This is because the code executes line by line. This is called sychronous code.

This creates lot of problems.For example, if a certain piece of code takes 10 seconds to execute, the code after it won't be able to execute untill it's finished, causing delays.


What is Asynchronous Code?

With asychronous code, multiple tasks can execute at the same time while tasks in the background finish. This is what we call nonblocking code. the execution of other code won't stop while an asychronous task finishes its work.


Let's see an example of asychronous code: 

        
            ?php
        $greet_one = "Hello";
        $greet_two = "world!!!";
        
        echo $greet_one . "\n";
        
        // Simulate asynchronous behavior with sleep()
        sleep(10); // Sleep for 10 seconds
        
        echo "Asynchronous\n";
        
        echo $greet_two . "\n";
        ?>

In the above example, if you run the code on your machine you will see greet _one and greet_two logged evenm there is code in between those 2 logs.

Now,set Timeout is asychronous so it runs in background, allowing code after it to execute while it runs. After 10 seconds, Asychronous will print because we set a time of 10 seconds in setTimeout to execute it after 10 seconds.

How callbacks work in PHP

"A callback function is a function passed into another function as an agreement, which is then invoked inside the outer function to complete some kind of routine or action". (MDN)

Let's look at a code example to see why using callbacks instead would be helpfull: 

    function compute(action, x, y){
        if(action === "add"){
            return x + y;
        } else if(action === "divide"){
            return x / y;
        }
    }
    echo(compute("add", 10, 5));
    echo(compute("divide", 10, 5));

In the above example,we have two operations.But what if we want to add more operations? Then the number of if/else statements would increase.This code would be lengthy,so we use callbacks instead: 

    
        ?php
    function add($x, $y) {
        return $x + $y;
    }
    
    function divide($x, $y) {
        return $x / $y;
    }
    
    function compute($callback, $x, $y) {
        return call_user_func($callback, $x, $y);
    }
    
    echo compute('add', 10, 5) . "\n";  // Output: 15
    echo compute('divide', 10, 5) . "\n";  // Output: 2
    ?>

Now when we call compute with three arguments,one of them is an operation. When we enter in the compute function,the function returns a funtion with a given action name.It, in response,calls that function and returns the result.

How promises work in PHP

  • A promse is a placeholder for the future result of an asynchronous operation.In simple words,we can say it is a container for a future value.
  • When using promises,we don't need to relay on callbacks which helps us avoid callback hell.
  • Before showing you how promises works through code,I'll explain promises using the analogy of a lottery ticket.
  • Promises are like lottery ticket.When we buy a lottery ticket, it says we will get money if our outcome is right.This is like a promise. The lottery draw happens asynchronously,and if the numbers match, we received the money which was promised.

Now let's look at a code example:

    ?php
        $url = 'https://course-api.com/react-store-products';
        $response = file_get_contents($url);
        
        // $response now contains the JSON data fetched from the URL
        echo $response;
        ?>

The above code is using fetch for a request from an API.It returns a promise which will get a response from the server.

This is how a promise looks. It has a particular promise state and result.When a promise is created it runs asynchronously.When the given task is completed,then we say the promise is settled.after the promise is settled,we can have either a fulfilled or rejected promise state based on outcome of the promise. We can handle these different states in different ways in our code.

How to Consume Promises

Consuming code is code that must wait for the result.so if we consume a promise, this ,means that when we make a request,we wait for the result.Then after result arrives,we perform some operation on those results. Let's continue using the above example to understand how we can consume a promise. We can consume promise using the then() method on the promise.Producing code is code that can take some time to complete. 

        
            ?php
        $url = 'https://course-api.com/react-store-products';
        
        // Fetch data from URL
        $response = file_get_contents($url);
        
        // Decode JSON response
        $data = json_decode($response);
        
        // Output the data (you can manipulate or use it as needed)
        var_dump($data);
        ?>
  • We make a request to the country API.Then,after the fetch request,we use the then() method to consume promise.After that,we return abunch of information like header,status and so on(you can see it in the below output image).
  • So we specifically need data which we need to convert to JSON which returns a promise. The data which s returned when we make a API request gets returned in the form of a promise. To handle that promise, we again use the then() method to log data from the response. Using multiple then() methods on a single request is called chaining promises.
